This site lets users browse public sync songs and, when logged in, publish or update songs they own. Review this draft with counsel before production use.

What data is used

Supabase handles email and password authentication. The app server stores song sync data, user profile defaults, and publishing metadata in PostgreSQL.

Profile defaults

Shortcut mappings, auto-fullscreen behavior, helper visibility, and other non-song defaults are stored in the authenticated user profile so they can follow the user across sessions.

Published songs

Published songs can remain visible after account deletion or closure. This is meant to keep public practice data intact, and the behavior should be clearly disclosed to users before they publish.

Future analytics and ads

Analytics and advertising features should stay disabled until the user gives consent. If Google Analytics or AdSense is added later, update this notice and the cookie notice before enabling them.

Your rights

Users should be able to request account deletion, ask questions about their data, and contact the site owner using the email above.
